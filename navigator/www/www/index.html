attendance logging


After the user login, it comes to the "attendance" page, whereby the system allow the user to click in check in date time and check out date time, by which the system is able to compute number of hours the volunteer has worked. it then retrieve from the previous record and add on to the "accumulation volunteered hours" and register this into the database Table Registration for individual, and Table attendance, which show a summary of this attendance logging.



computing and updating of the accumulated volunteered hours


The first field in the attendance loggin form, aVolHoursPrevious, will be extracted from the database registration table.

Then user input with checkin checkout DTS (or can be improved as a QR code checkInDTS checkOutDTS scan late if necessary),

the attendance.js will compute the Volunteered Hours worked (i.e., volHours). new aVolHours = aVolHoursPrevious + volHours this new aVolHours will (1) be displayed on the forth field (after submit button)

(2.i) update the table attendance.aVolHoursPrevious and (2.ii) replace the first field on the aVolHoursPrevious = new aVolHours.

(3) update the table registration.aVolHours for this user

input validation




The app has a validation that the check out DTS cannot be earlier than the check in DTS, to assist user avoid error input.

offline input accessible


This case report form facility is located "outside" the "login" page. This enables the public to input for reporting a case upon emergency without being a registered volunteer.

The public can scan QR code at the entrance to the track and get this app on their smart phone. Alternatively, the QR code can be placed together with the Waugal Trail Marker whereby the internet connection is available.

Instead of only relying on registered volunteers, this mobilise the public initialtive to report whatever needed especially after a natural disaster .


Offline GPS

GPS data can be captured using the Geolocation API and stored in localStorage (inside user's handphone).

GPS extract latitude and longitude directly from satellite and hence it does not need to interact with internect connection.

Camera Video

Camera and video data can be captured using the MediaDevices API and stored as " base64 strings " in localStorage.

Technology

When the user performs actions like clickGPS, capturing images or recording Video, this app convert the data into a format suitable for localStorate (JSON).

There are 2 buttons at the end of this page (see bottom right corner of this scren shot):

(1)The first button click to store data on localStorage (handphone or tablet devices).

(2)The second button click to

(2.i) synchronise and transfer all form data including GPS and camera video to databases when internet connection is available,
(2.ii))with the automation to clear (get ready for next saving) localStorage on the smart phone.


date time picker highlight


the picker automatically bounce out with the current time. This reduces chance of user's typo error input. (If necessary,) this feature can also be replaced by "current-date-time insertion" so user need not even specific date time.

ease of selection from the 55 campsites


There are 2 options to select the campsite indicator.

OPTION A:

a.i. When user select DEC district, the repsective campsites belonged to the particular selection DEC district authority will emerge for further selection, This reduces chance of user's typo error input.

a.ii. the selected DEC district comes with a pre-built email address of this officer in the DEC district office. When the user select "Yes" for "email DEC?", the last button will send email of the entire report to the DEC district officer automatically when it synchronise the data to database with internet connection.

OPTION B:

b. the second option, on the last selection of the "DEC district" dropdown, is the "all campsites", where, a complete list of all bibbulmun track given by Terry's email was systematically organised in alphephatical order, eliminated the duplicates.


Offline GPS indicator of exact location


GPS extract latitude and longitude directly from satellite without internet connection. This enable an accurate indication of the location for easy tracing or salvation if needed. Additionally, this GPS can be also be use as a good "SOS" location indicator if the "victims" are able reach some place with internet connection.


camera and video features


The app comes with the camera and video features to convenient the user in saving the images in their hand phone, and synchronise to database and send out email (if necessary) when reaching a point with internet connection.

OFFFLINE Save Report to Device


At the end of the report, if it is offline, without internet connection, the user click button "OFFLINE Save Report to device", the report wil be saved on their handphone.


ONLINE Data Sync to Database and Send Email to DEC district


When the internet connection is available

(1) the user can click "ONLINE Data Sync to Database Email DEC" button, the report information will be transfer to databases, with an option to remove them from the handphone device temporary storage.

(2) At the same time, if the user select "Yes" for "email to DEC", the report wil be email to the respective district officer automatically.
